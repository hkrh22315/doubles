# コードレビュー

## クイックスタート

コードをレビューするとき:

1. 正しさと潜在的なバグを確認する
2. セキュリティのベストプラクティスを確認する
3. 可読性と保守性を評価する
4. テストが十分か確認する

## レビューチェックリスト

- [ ] ロジックが正しく、エッジケースを扱えている
- [ ] セキュリティ脆弱性がない（SQL インジェクション、XSS、資格情報露出、API キー漏えい）
- [ ] プロジェクトのスタイル規約に従っている
- [ ] 関数の粒度が適切で責務が明確
- [ ] エラーハンドリングが網羅的
- [ ] 変更点がテストでカバーされている
- [ ] 外部 API・リソースの利用がレート制限・エラー処理・設定の検証の面で妥当か

## フィードバックの書き方

重要度（Severity）で分けて書く:

- **Critical**: マージ前に必ず修正が必要
- **Suggestion**: 改善を検討
- **Nice to have**: 任意の改善
