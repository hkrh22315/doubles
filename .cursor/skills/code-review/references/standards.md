# コードレビュー追加基準

どのプロジェクトにも適用できる追加レビュー基準。

## 資格情報の取り扱い

- API キーやシークレットは環境変数（例: `XXX_API_KEY`, `XXX_API_SECRET`）等で読み込み、コードへハードコードしない
- 設定ファイル（YAML 等）に実際の資格情報を入れない（プレースホルダ、または「環境変数を使う」旨のコメントにする）
- 資格情報（全部または一部）をログ出力・print しない

## 外部 API・リソースの利用

- 外部 API のレート制限や連続エラー上限の扱いが適切か確認する
- API 呼び出し前に、パラメータや設定がサポート範囲内か検証する
- 利用量・サイズが設定上の制約を満たすことを確認する

## ログ・エラーハンドリング

- 重要なオペレーションは主要イベントをログに残す
- 外部 API 等のエラーは捕捉し、ログに残し、適切に処理する
- ログメッセージに機密情報を含めない
- ログレベルを適切に使い分ける（冗長: DEBUG、運用イベント: INFO、失敗: ERROR）
